## 摘要
尽管大规模预训练彻底改变了语言建模，但在使用结构化电子健康档案（EHR）的医疗保健领域，其潜力仍未得到充分挖掘。我们提出了 RAVEN，这是一种基于循环感知下次就诊事件预测的序列化 EHR 数据生成式预训练策略。利用超过一百万名独立个体的资料集，我们的模型学习在给定患者历史记录的情况下，自回归地生成下次就诊的标记化临床事件。我们引入了针对重复事件预测的正则化，并指出了基于 EHR 的基础模型评估中的一个关键陷阱：当新发病例与后续发生未加区分时，重复的事件标记会虚高性能指标。此外，我们实证研究了在数据受限、计算饱和状态下的扩展行为，结果表明，在没有相应增加数据量的情况下，单纯增加模型规模并非最优选择。我们通过零样本预测评估了模型对多种疾病发病率的预测能力，其表现可与完全微调的基于表示的 Transformer 模型相媲美，并优于广泛使用的基于模拟的下个标记方法。最后，在无需额外参数更新的情况下，我们证明了 RAVEN 在存在有损临床代码映射和特征覆盖缺口的情况下，仍能泛化到外部患者队列。

## Abstract
While large-scale pretraining has revolutionized language modeling, its potential remains underexplored in healthcare with structured electronic health records (EHRs). We present RAVEN, a novel generative pretraining strategy for sequential EHR data based on Recurrence-Aware next-Visit EveNt prediction. Leveraging a dataset of over one million unique individuals, our model learns to autoregressively generate tokenized clinical events for the next visit conditioned on patient history. We introduce regularization on predicting repeated events and highlight a key pitfall in EHR-based foundation model evaluations: repeated event tokens can inflate performance metrics when new onsets are not distinguished from subsequent occurrences. Furthermore, we empirically investigate the scaling behaviors in a data-constrained, compute-saturated regime, showing that simply increasing model size is suboptimal without commensurate increases in data volume. We evaluate our model via zero-shot prediction for forecasting the incidence of a diverse set of diseases, where it rivals fully fine-tuned representation-based Transformer models and outperforms widely used simulation-based next-token approaches. Finally, without additional parameter updates, we show that RAVEN can generalize to an external patient cohort under lossy clinical code mappings and feature coverage gaps.

*   **背景挑战**：
    1.  **就诊内无序性**：单次就诊内的临床事件（诊断、用药、化验）通常没有严格的先后顺序，传统的“下一标记预测（Next-token prediction）”并不适用。

    3.  **数据规模限制**：医疗数据远少于互联网文本，盲目增加模型参数可能导致严重的过拟合。

#### 2. 方法论：核心思想与技术细节
论文提出了 **RAVEN**（Recurrence-Aware next-Visit EveNt prediction）框架：
*   **核心思想**：将预训练目标设定为“预测下一次就诊的完整事件集”，并引入针对重复事件的惩罚机制。
*   **关键技术细节**：
    *   **下次就诊多标签预测**：模型在每个就诊结束的 `[SEP]` 标记处，预测下一次就诊中所有可能出现的临床标记（多标签分类），避免了对就诊内事件进行人为排序。
    *   **循环感知正则化（Recurrence-Aware Regularization）**：引入衰减因子 $\lambda$。对于目标事件 $k$，根据其在患者历史中出现的次数 $c$ 下调其损失权重：$w = \max(\lambda^c, w_{min})$。这迫使模型将注意力从“重复已知疾病”转移到“预测新发疾病”上。
    *   **架构改进**：采用 Decoder-only Transformer，修改注意力掩码以允许就诊内标记互视，但保持就诊间的因果序；使用旋转位置嵌入（RoPE）编码从首次就诊起的绝对时间。
    *   **零样本推理接口**：通过在输入序列末尾添加带有未来时间戳（如 $t+5$ 年）的 `[SEP]` 标记，单次前向传播即可获得未来风险评分。

#### 6. 主要结论与发现
*   **零样本能力强**：RAVEN 在零样本下的表现优于所有基于模拟的自回归基准，且在多个任务上接近甚至超过了完全微调的监督模型。
*   **正则化的必要性**：引入循环感知正则化显著提升了模型预测“首次发病”的准确性（准时率翻倍），尽管这可能会略微降低整体 AUPRC。
*   **计算饱和下的扩展规律**：在 EHR 这种数据受限场景下，模型性能随参数增加呈 **U 型曲线**（即过大的模型会因重复训练而过拟合）。模型容量必须与可用患者数量精确匹配。
*   **跨机构泛化**：尽管存在代码映射损失（仅利用了 25% 的特征），RAVEN 仍能在外部数据集上达到与少样本（Few-shot）监督模型相当的水平。

#### 7. 优点（亮点）
*   **评估洞察**：深刻揭示了 EHR 建模中“重复标记”导致的评估偏差，并提出了有效的正则化解决方案。
*   **推理效率**：相比需要多次模拟（Rollouts）的传统生成模型，RAVEN 的单次前向传播推理速度极快，更适合临床大规模筛查。
*   **扩展性研究**：不同于互联网数据的无限假设，本文研究了在“有限数据、无限算力”这一更符合医疗现实的情况下的模型扩展行为。

#### 8. 不足与局限
*   **特征覆盖缺口**：在外部迁移时，由于不同医院编码体系差异，导致大量特征（如生命体征）被丢弃，限制了泛化性能。
*   **代理指标限制**：EHR 中的记录时间仅是临床发病的代理（Proxy），可能存在滞后，模型无法完全消除这种系统性偏差。
*   **干预性缺失**：模型目前仅能做观察性预测，尚无法模拟“如果给予某种治疗”后的干预性结果（Counterfactual reasoning）。
*   **数据单一性**：预训练主要基于单一大型医疗系统，不同地区的编码习惯差异可能影响模型的普适性。
