## 摘要
工业金融系统运行在诸如交易、用户行为和系统日志等时间事件序列之上。尽管最近的研究强调表示学习和大型语言模型，但由于可解释性、有限监督下的鲁棒性以及严格的延迟限制，生产系统仍然严重依赖手工制作的统计特征。这导致了学习到的嵌入（embeddings）与基于特征的流水线之间存在持久的脱节。我们提出了嵌入感知特征发现（EAFD），这是一个统一的框架，通过将预训练的事件序列嵌入与自我反思的 LLM 驱动特征生成代理相结合来弥合这一差距。EAFD 利用两个互补的标准直接从原始事件序列中迭代地发现、评估和优化特征：一是“对齐性”（alignment），用于解释已编码在嵌入中的信息；二是“互补性”（complementarity），用于识别嵌入中缺失的预测信号。在开源和工业交易基准测试中，EAFD 始终优于仅使用嵌入和基于特征的基准模型，在最先进的预训练嵌入基础上实现了高达 +5.8% 的相对提升，从而在事件序列数据集上创造了新的最先进性能（SOTA）。

## Abstract
Industrial financial systems operate on temporal event sequences such as transactions, user actions, and system logs. While recent research emphasizes representation learning and large language models, production systems continue to rely heavily on handcrafted statistical features due to their interpretability, robustness under limited supervision, and strict latency constraints. This creates a persistent disconnect between learned embeddings and feature-based pipelines. We introduce Embedding-Aware Feature Discovery (EAFD), a unified framework that bridges this gap by coupling pretrained event-sequence embeddings with a self-reflective LLM-driven feature generation agent. EAFD iteratively discovers, evaluates, and refines features directly from raw event sequences using two complementary criteria: \emph{alignment}, which explains information already encoded in embeddings, and \emph{complementarity}, which identifies predictive signals missing from them. Across both open-source and industrial transaction benchmarks, EAFD consistently outperforms embedding-only and feature-based baselines, achieving relative gains of up to $+5.8\%$ over state-of-the-art pretrained embeddings, resulting in new state-of-the-art performance across event-sequence datasets.

*   **核心问题**：在工业生产环境中，尽管深度学习生成的嵌入（Embeddings）表现优异，但由于可解释性、鲁棒性及低延迟要求，系统仍高度依赖手工制作的统计特征。
*   **研究动机**：目前的嵌入表示存在“盲点”（即无法捕捉某些关键的统计或时间模式），而现有的自动特征工程（AutoFE）工具通常独立于嵌入运行，导致特征冗余或无法弥补嵌入的缺陷。论文旨在建立一个桥梁，让 LLM 驱动的特征生成能够感知嵌入的不足，从而实现两者的协同优化。

### 2. 方法论：EAFD 框架
EAFD 是一个迭代的、自我反思的框架，核心思想是利用 LLM 代理（Agent）根据预训练嵌入的状态来发现新特征。
*   **核心流程**：
    1.  **嵌入锚定**：使用冻结的预训练编码器（如 CoLES, NTP）生成序列嵌入。
    2.  **LLM 特征生成**：LLM 代理根据原始序列数据和之前的反馈，提出可执行的 Python 代码来定义新特征。
    3.  **双重评估准则**：
        *   **对齐性（Alignment）**：衡量特征与嵌入的相关性（使用 $R^2$ 分数）。高对齐性意味着特征解释了嵌入中已有的信息。
        *   **互补性（Complementarity）**：衡量特征对下游任务的增量贡献。高互补性意味着特征捕捉到了嵌入的“盲点”。
    4.  **自我反思与修正**：如果代码运行失败，框架会触发调试模式；如果特征效果不佳，LLM 会根据反馈在下一轮迭代中调整策略。

*   **盲点识别**：发现 CoLES 等模型在捕捉时间动态和数值分布（如交易金额）方面存在系统性缺陷。
*   **多目标适应性**：在工业数据集中，EAFD 能根据不同任务（年龄 vs 性别 vs 金融预测）自动调整特征生成的侧重点（如年龄预测侧重活动模式，回归任务侧重时间特征）。

### 7. 优点与亮点
*   **闭环优化**：首次将 LLM 的推理能力与嵌入空间的定量分析（对齐与互补）结合，实现了“嵌入感知”的特征工程。
*   **工业实用性**：生成的特征是 Python 代码，具有高度的可解释性和可移植性，符合生产环境的需求。
*   **诊断功能**：不仅是提升性能的工具，还能作为预训练模型的诊断工具，指导编码器架构的改进。

### 8. 不足与局限
*   **计算成本**：依赖大型 LLM（120B 参数）进行多次迭代生成，对于算力受限的环境可能成本过高。
*   **延迟风险**：虽然生成的特征在推理时很快，但特征发现过程（搜索空间）可能较慢。
*   **LLM 偏差**：特征的质量高度依赖于 LLM 的先验知识和代码生成能力，对于极度冷门的领域，LLM 可能无法提出有效的特征逻辑。
*   **数据依赖**：实验主要集中在金融交易领域，在其他类型的事件序列（如传感器数据或生物信息）上的泛化能力尚待验证。
